Getting started

  1. Install. Unzip the pack for your machine and run the installer inside — install.bat on Windows, install.command on a Mac. No admin rights needed; it only registers the add-in for your own account.
  2. Restart Word completely. Close every window, not just the document. Word only notices a new add-in on a cold start — this is the single most common reason it "didn’t install".
  3. Open the pane. Insert → Add-ins → Developer Add-ins → JurisLab. It opens on the Home screen showing all 24 tools as cards.

Only the first load needs internet — the add-in fetches itself from the web once, then everything you type is computed on your own machine.

The core loop

Every tool works the same way, so once you’ve learned one you’ve learned all of them:

  1. Put your cursor where you want the content to land in your document.
  2. Pick a tool — click a card on the Home screen, or use the dropdown at the top.
  3. Type. A live preview appears as you go, showing exactly what will be inserted.
  4. Insert. The button drops it at your cursor. If it isn’t what you wanted, a single Ctrl-Z (⌘-Z on a Mac) removes it cleanly.
Try it: open Chemical, type H2O → the preview shows H₂O → Insert formatted text. Then type aspirin → a 2D structure appears with its formula and molecular weight.

Chemistry & structures

Chemical — formulas and 2D structures

Type a formula and it typesets itself: H2O → H₂O, Ca(OH)2 → Ca(OH)₂, SO4^2- → SO₄²⁻, Na+ → Na⁺. Type a name (caffeine), a formula (C6H6) or any SMILES and you get a 2D structure, plus physicochemical properties (cLogP, tPSA, Lipinski/Veber druglikeness) and a pKa estimate with the net charge at pH 7.4.

For stereochemistry, enter an isomeric SMILES — C[C@@H](N)C(=O)O — and it’s drawn with wedge and hash bonds.

Build — structures from atoms and bonds

Try it: atoms: C O O then bonds: 1=2 1=3 → CO₂. The palette buttons insert templates (benzene, acetic acid) and Markush tokens for you.

Also handles Markush / genus structures for patent work: [C,N] variable atoms, X halogen, R1 R-groups, plus query features like {ar} (aromatic) and {r6} (ring size 6).

Mass Spec

From a name, formula or SMILES: exact monoisotopic and average mass, the theoretical isotope pattern, and common adduct m/z ([M+H]⁺, [M+Na]⁺, [M−H]⁻ …).

Try it: caffeine → monoisotopic 194.0804, [M+H]⁺ 195.0877. A chlorinated compound shows the tell-tale ~32% M+2 peak.

Spectra — predicted NMR, IR, UV-Vis & fragmentation

Pick a spectrum type, type a name or SMILES, and get a prediction with an assignment for every signal. Insert it as a data table or a publication-ready chart.

Try it: toluene, ¹H NMR → δ 7.17 (2H, m), 7.09 (2H, d), 7.08 (1H, t), 2.37 (3H, s, CH₃ on Ph).
Read this before using Spectra. These are estimates from published additivity rules — not measured spectra, and not quantum-chemical calculations. The structure recognition is exact, but the numbers carry real uncertainty (about ±0.2–0.4 ppm for ¹H, ±2–4 ppm for ¹³C). The tool tells you where it’s weak rather than hiding it: fused and heteroaromatic rings are flagged approximate, an unconjugated molecule reports no λmax instead of inventing one, and MS “likelihood” is a ranking, not an intensity. Use it to assign peaks you already have — never as a substitute for running the spectrum.

Reaction

Multi-step schemes with conditions over the arrow: CCO + CC(=O)O >> CC(=O)OCC ; H2SO4 ; reflux.

Numbers & analysis

Analyze — a no-code numerical workbench

Paste numbers, get real computed results and a figure, without writing any code:

  • Matrix maths — solve A·x = b, inverse, determinant, rank, trace, eigenvalues (including complex), QR and SVD. Results insert as editable Word tables.
  • Matrix expressions — one line: A*inv(B)+2*C'.
  • Optimization — minimise a function you type.
  • FFT — amplitude spectrum and dominant frequencies from a sampled signal.
  • Data → insights — paste a table and get column summaries, correlations (with p-values), trends, outlier flags, and a plain-language write-up.

Analyze → Solve an ODE

Type the equation you actually have. Higher-order equations are reduced for you — you do not hand-convert to a first-order system.

Try it: equations y'' = -y, initial values y = 1, y' = 0, range 0, 6.2832 → solves and reports “auto-reduced to a first-order system of 2 states”, ending at y = 1 (it’s cos t).
FieldWhat to put in it
EquationsOne per line. y' = -y, or y'' = -0.1*y' - y, or a system. Right-hand sides can use t, the state names, and functions like exp, sin, tanh, sqrt, min/max, mod, and if(cond, a, b) for switching inputs.
Initial valuesOne per state: y = 1, y' = 0. A second-order equation needs two.
Report at timesOptional. A list (0, 1, 2.5) or a range (0:0.5:10). Those values are computed exactly, not interpolated. Leave blank to get the solver’s own steps.
Stop whenOptional. An expression that ends the solve where it crosses zero — answers “when does it…?”
SolverLeave on Auto. It starts with a fast explicit method and switches to an implicit one if the problem turns stiff — which is normal for kinetics whenever rate constants differ by orders of magnitude.
“When does it land?” Equations z'' = -9.81, initials z = 0, z' = 20, range 0, 10, Stop when z → “z reached zero at t = 4.077472”, and the curve ends at the ground.

Stats

Descriptive statistics, t-tests (Welch/Student/paired), one- and two-way ANOVA, regression, non-parametric tests (Mann–Whitney U, Wilcoxon), chi-square, multiple-comparison correction (Bonferroni/Holm/Benjamini–Hochberg), and uncertainty propagation.

Finance

18 calculators — time value of money, loans, NPV, IRR, DCF, Black–Scholes with Greeks, bond analytics — plus a finance equation library for typesetting the formulas themselves.

Math, units & figures

Math — native Word equations

Type x^2 + y^2 and insert a real Word equation (click it and Equation Tools appear — it’s editable, not a picture). Fractions, roots, Σ, ∫, matrices, piecewise cases(…), Greek, and multi-line align(…).

Coming from LaTeX? Paste \frac{-b \pm \sqrt{b^2-4ac}}{2a} and convert it to a Word equation — or copy what’s in the box back out as LaTeX.

Tick Number this equation for an auto-numbered (I), (II)… The Formula library dropdown has ready-made expressions across maths, physics, engineering and finance.

Units

SI typesetting (9.81 m/s^2 → 9.81 m/s²), uncertainties (5.0 +- 0.2 kg → 5.0 ± 0.2 kg), significant figures, and conversion including compound units (36 km/h → 10 m/s). Incompatible conversions are refused, not fudged.

Plot

Function and data charts, multiple series with a legend, error bars — computed and drawn offline.

Table → Chart

Put your cursor in a Word table of numbers and turn it into a chart (column, bar, line, area, scatter, stacked), a flowchart, or a block diagram. Includes a black-and-white patent-figure mode, and can produce an editable PowerPoint instead of an image.

Code

Algorithm listings with bold keywords and numbered lines, or verbatim code blocks.

Life sciences

Bio/Assay

Offline curve fitting with parameters ± standard errors and R², and the fit drawn over your data: enzyme kinetics (Michaelis–Menten, Hill), dose–response (IC50/EC50, 4PL), one-site binding, plus lab maths (Cheng–Prusoff, Henderson–Hasselbalch, Beer–Lambert, dilutions, A260/A280).

Peptide

A 2D structure from a one- or three-letter sequence (AGCW or Ala-Gly-Cys), with formula and molecular weight.

DNA

Reverse complement (RNA-aware), transcription, six-frame translation, ORF finder, primer Tm, protein MW/pI/GRAVY, and restriction sites.

Sequence Map

Open a sequence file and insert an annotated map at the cursor. Read entirely on your machine; nothing is uploaded.

FormatWhat you get
GenBank .gb .gbk .gbffThe one that draws a real map, because it carries the feature annotations — promoters, CDS, resistance markers, primer sites, origins.
FASTA .fa .fastaSequence only. FASTA has no annotations and no topology, so you get the backbone and scale but no features. That is the format, not a limitation of the tool.
SnapGene .dnaRead where possible; it falls back to GenBank with a message rather than guessing.

Plasmids draw as circular maps, linear constructs as linear ones, and Auto picks from the file’s own topology. Restriction sites can be overlaid.

Methylation changes what actually cuts. The site list is what the enzyme recognises; whether it cuts depends on how the DNA was grown. MboI will not touch plasmid DNA from an ordinary dam+ strain, and DpnI cuts only methylated DNA. JurisLab flags both under the digest table — read it before you set the reaction up.

Align

Compare two sequences and insert the alignment at the cursor. Needleman–Wunsch (global) and Smith–Waterman (local) with affine gaps — the same algorithms EMBOSS needle and water run, at their default costs (BLOSUM62, gap open 10 / extend 0.5 for protein; +5/−4 for DNA). Paste plain sequence or FASTA. Everything runs on your machine.

Global forces an end-to-end alignment — use it when the two sequences are the same thing: your clone against the reference, two alleles, a read against the expected insert. Local finds the best shared segment — use it when they share only a domain, or differ a lot in length.

Percent identity is not a fact about the sequences. It is a fact about the alignment, and it moves when you change the gap costs. Below about 25% identity (the “twilight zone”) two completely unrelated proteins will still align convincingly — JurisLab says so when you get there. An “optimal” alignment is the highest-scoring one under the parameters you chose, not necessarily the biologically correct one.

The alignment inserts in a monospace font deliberately: the ruler’s | must sit directly above the residue it refers to, and a proportional font breaks that while still looking like an alignment.

Sequence

WIPO ST.26 biological sequence listings (DNA/RNA/protein) with CDS and gene annotation.

Always validate ST.26 output in the WIPO Sequence tool before filing. JurisLab generates the listing; it is not a substitute for WIPO’s own validator.

Botanical

Plant-patent scientific-name typesetting (Rosa × hybrida 'Peace' → Rosa × hybrida 'Peace') and varietal trait tables.

Patent & legal

Citations

Bluebook citations for cases, statutes, patents, the Federal Register and the MPEP, in practitioner or academic style, with T6/T10 abbreviation. Builds a Table of Authorities from the citations in your document and a Table of Contents from its headings.

Numerals

Reference-numeral management: a numeral → element table saved in the document, callouts inserted at the cursor, and a document scan that flags collisions (one numeral, two elements), gaps, orphans (a callout with no entry) and unused entries. Inserts the sorted List of Reference Numerals.

Refs

Auto-numbered figure and table captions with cross-references.

Audit

One pass over the whole document checking everything at once: reference numerals, SEQ ID NO references against the listing, figure numbering, and dangling cross-references. A clean document reports “✓ No issues found.”

Honest limits — what JurisLab does not do

Knowing where a tool stops is what makes the rest of it trustworthy.

  • It is a drafting aid. Verify every structure, formula, calculation and listing before filing or publishing. It is not a reviewer.
  • Predicted spectra are estimates, not measurements — see the warning in the Spectra section above.
  • pKa is a functional-group estimate, not a compound-specific prediction. The group detection is exact; the value is a typical literature figure for that group.
  • The ODE tool is numerical only — no symbolic/closed-form solutions, no boundary-value problems, no PDEs.
  • IR predicts group frequencies, not the fingerprint region below ~1500 cm⁻¹ — that’s compound-specific and no rule reproduces it.
  • UV-Vis uses the Woodward–Fieser rules, which have a defined domain (conjugated dienes and α,β-unsaturated carbonyls). Outside it, JurisLab says so instead of guessing.
  • ST.26 listings need WIPO validation before filing.
  • Word on the web / older Word: native equations need a recent Word. Where they aren’t available, JurisLab inserts formatted text instead and tells you it did.

Troubleshooting

SymptomWhat to do
The add-in doesn’t appear after installingFully close and reopen Word — all windows, not just the document. This is the usual cause. Then look under Insert → Add-ins → Developer Add-ins.
The pane is blank or won’t loadYou need internet for the first load. If it’s still blank, close the pane and reopen it. If it persists, reinstall from the current pack — an older install can point at an address that no longer exists.
A banner says a new version is availableReload the pane (close and reopen it). You don’t need to reinstall — the add-in updates itself from the web.
“Couldn’t build an equation from that”The expression didn’t parse, so it inserted formatted text instead. Check for an unclosed bracket. The Examples & syntax panel shows the syntax for what you’re typing.
The preview shows an errorTrust it — that’s the preview doing its job. Nothing is inserted until you click Insert.
An ODE says it stalled or looks stiffSet the solver to Auto (or Implicit). If it still won’t finish, the solution may genuinely blow up in finite time — try a shorter range.
A structure name isn’t recognisedTry its SMILES, or enable the opt-in online IUPAC name lookup when prompted.
Something inserted that you didn’t wantCtrl-Z (⌘-Z on a Mac). Every insert is a single undo step.
